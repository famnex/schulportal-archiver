# Database Documentation

This document describes the structure of the SQLite database used by the Schulportal Hessen Archiver.

## Schema Overview

The database contains a `users` table that stores the encrypted SPH credentials of the user.

### Table: `users`

Stores credentials securely encrypted with AES-256-GCM.

| Column | Type | Constraints | Description |
| :--- | :--- | :--- | :--- |
| `id` | `INTEGER` | `PRIMARY KEY AUTOINCREMENT` | Auto-incrementing identifier. |
| `school_number` | `TEXT` | `NOT NULL` | The 4-digit Schulnummer (e.g., `9743`). |
| `username` | `TEXT` | `NOT NULL` | The login username (e.g., `max.mustermann` or `sfl`). |
| `encrypted_password` | `TEXT` | `NOT NULL` | The AES-256-GCM encrypted password in hex format. |
| `iv` | `TEXT` | `NOT NULL` | The initialization vector used for encryption (12 bytes, hex format). |
| `auth_tag` | `TEXT` | `NOT NULL` | The 16-byte authentication tag generated by AES-GCM (hex format). |
| `created_at` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | The timestamp when the record was created. |
| `updated_at` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | The timestamp when the record was last updated. |

* **Constraints:**
  * `UNIQUE(school_number, username)`: Ensures a user can only have one password stored per school.

## Schema Initialization Query

```sql
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    school_number TEXT NOT NULL,
    username TEXT NOT NULL,
    encrypted_password TEXT NOT NULL,
    iv TEXT NOT NULL,
    auth_tag TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(school_number, username)
);
```
